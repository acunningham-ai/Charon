It's just files

Yours the day you walk away from the tool.

Every memory is a plain markdown file with frontmatter and wiki-links. No database to trust, no opaque store — your knowledge stays in the open, versionable, portable, and yours the day you walk away from the tool.

A one-line index (MEMORY.md) points at each file, so the model knows what exists without loading everything at once.

--- type: project · status: active ---
Renew wildcard cert before 2026-09-14.
Feeds [[q3-audit]] · owner [[platform-team]]
→ q3-audit→ platform-team→ renewals

Save on mention

Spoken once. Written before the turn ends.

When you state a fact worth keeping, it's classified and written to the right file in the same reply. A second-stage model decides where it belongs — and closed enums keep the frontmatter valid instead of free-form drift.

Known secret patterns — API keys, tokens, passwords — are redacted before anything is sent to the classifier. Credentials never enter the memory files; they stay in your secrets directory, pointed at, never quoted.

you ▸ the bot restarts need both services, not just one # save-on-mention ✓ routed → project CLAUDE.md (operational fact) ✓ secrets scan (none present) → "saving the restart sequence now"

Optional layer

A typed graph, when you want relationship queries.

The default is plain markdown — and that's enough for recall. If you want to traverse your knowledge, an optional graph layer adds typed entities and relationships on top of the same files.

opt-in · requirements-graph.txt

Ask what connects to what.

Install the optional graph dependencies and a background extractor builds a typed entity + relationship graph from your notes. Then query it in plain language — shortest path between two people, everything touching a project, a neighbourhood around one entity.

networkx-backed, stored as JSON — no native build step. Read-only queries through the vault-graph server and /vault-query. Leave it uninstalled and memory still works exactly the same.

The point

Your memory outlives the session — and the tool.

It's .md files in a folder. Read them, edit them, grep them, put them in git. Charon loads them; it doesn't lock them up.
