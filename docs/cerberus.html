Cerberus

Defensive security for the AI installation itself.

The review skills check the code you write. Cerberus checks the installation around it — the config, the plugins, the MCP servers, and the dependencies you pull in. Most AI-security tooling that shipped in 2025–26 is offensive; the defensive surface has been under-tooled, and that's what Cerberus covers.

Example session — real capability, illustrative data.

Vet it before you install it.

/cerberus-vet <repo-url> risk-assesses a third-party plugin, skill, or MCP server before it touches your setup — clones to a throwaway sandbox (purged after), scores it against a V0–V8 threat model, and returns a level (LOW / MEDIUM / HIGH / CRITICAL) with per-finding remediation.

Output is risk evidence, not approval — the final call sits with your organisation's policy. Read-only; no auto-installs.

# /cerberus-vet github.com/example/some-mcp clone → sandbox (purged after) V2 network egress ⚠ MEDIUM LLM03 supply chain V5 md injection ✓ none V6 secret exposure ✓ none V8 deps vs registry ✓ clean → risk MEDIUM · 58/100 — evidence, not approval

Five commands

Set up, audit, vet, check deps, recover.

/cerberus-setup

First-run hardening wizard — audits your setup against the gold standard, walks each gap, verifies. The first thing you run on a new machine.

/cerberus-audit

Read-only audit across the 7-layer threat model → a 0–100 score and per-finding fixes. Run monthly, or after any change to settings/plugins.

/cerberus-vet

Pre-install risk assessment of a third-party plugin / skill / MCP against V0–V8. Risk level + score; evidence, not approval.

/cerberus-deps

Audit your own project's dependencies against a live compromise registry (LiteLLM 1.82.7/8, telnyx, tiledesk, typosquats, Shai-Hulud). Reports hits + suggested pins.

/cerberus-recover

Post-leak runbook — rotation, git-history cleanup, session invalidation, hardening. Run the moment you suspect a secret was exposed.

Grounded in published standards

Every check cites OWASP, not internal opinion.

The V0–V8 vetting layers map to specific entries in the OWASP Top 10 for LLM Applications (2025), so findings trace to a recognised industry frame.

Secure-by-design, not retrofitted: read-only across all commands, sandbox-disciplined cloning, and a validation-honest field on every finding (theoretical / partial / validated) — nothing claims "validated" unless a proof-of-concept actually ran. Hook scripts ship available-but-not-auto-wired — opt-in, never silently enabled. Original engine by Joh Leonhardt (MIT).
