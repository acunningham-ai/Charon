Workflows

Fan out in parallel. Verify before you trust.

Workflows are deterministic orchestration scripts that spread a hard problem across many agents, then converge. Their defining move: a finding isn't trusted because an agent produced it — it's challenged by independent verifiers, and only survives if it holds.

The pattern · loop, not line

Every finding faces a skeptic whose job is to refute it.

A single agent producing an answer is a guess with good grammar. So each surfaced finding is handed to independent verification passes — prompted to break it, not confirm it. Majority-refute kills the finding. What's left has survived an argument.

Honest scope: this is one model casting several independent votes on the same claim — not a panel of different vendors' models. It catches over-confidence and weak evidence; it isn't model-diversity.

What ships

Two workflows, both self-verifying.

Each is a JavaScript file you can read in .claude/workflows/. They spawn many agents and use real tokens — built for genuinely hard questions and high-stakes calls, not quick asks.

Deep research

/deep-research returns · a cited report

Decompose a question into search angles, fan out parallel web search, fetch and extract falsifiable claims — then a 3-vote adversarial verify. Evidence-handling rejects are re-researched against a live source until the verify pass finds nothing left to re-queue.

Devil's advocate

/devils-advocate returns · kill / proceed-with-fixes / proceed

An adversarial pre-mortem for hard-to-reverse non-code decisions — a hire, a launch, a policy line, a big bet. A framing gate first checks the decision is sharp enough to test; then it fans out hostile lenses alongside one steelman making the honest case for, consolidates, and 3-vote verifies every surfaced risk — flagging any load-bearing risk the skeptics kill unanimously for a second look — then a grounding gate tags each survivor grounded, plausible, or invented. Draft-only — it writes nothing.

Safety posture

Read and reason. Nothing else.

A workflow spawns a lot of agents. Here's what it's allowed to do with them.

Read + reason only

Neither workflow writes to your vault, sends, or posts. Deep research returns a report; devil's advocate returns a verdict. Any save is a separate, human-confirmed step.

Untrusted input is data

Anything an agent reads — your input, a fetched page, a vault note — is wrapped as data, not instructions. An embedded "ignore your instructions" becomes a finding, not a command.

No capability creep

Grounding and verify agents are read-only by instruction; the orchestration writes nothing. The scale is in the reasoning, not in what it can touch.
