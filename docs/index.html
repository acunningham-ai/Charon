Durable memory

What you tell it today loads automatically tomorrow.

Claude Code starts every session from zero. Charon keeps a memory as plain markdown you own — written the moment you mention something, loaded when it's relevant.

1

You say it

Drop a fact, decision, person or project into the conversation — no special syntax.

2

Saved the same turn

The save-on-mention hook routes it to the right file and acknowledges — not batched.

3

Loaded next time

Name a project or person next session and the relevant memory loads first.

It's just files.

Every memory is markdown with frontmatter and wiki-links — readable, grep-able, portable, and yours the day you walk away from the tool. An optional typed graph adds relationship queries on top.

Default is plain markdown — enough for recall. The graph is opt-in.

--- type: project · status: active ---
Renew wildcard cert before 2026-09-14.
Feeds [[q3-audit]] · owner [[platform-team]]
→ q3-audit→ platform-team→ renewals

Automatic capture

Your world flows in — you don't type it twice.

A working pipeline pulls your email — inbox and sent — into the vault as markdown, on your schedule. Because it captures what you sent too, it can surface "you replied to Karen two days ago, still no answer" — time-management signal an inbox-only tool can't produce.

Every capture is marked untrusted and read as data, never instructions. When company policy expires the sign-in, it fails fast with a desktop toast and a one-line recovery banner next session — no silent morning failures. M365 ships fully working; Gmail/IMAP are documented skeletons.

# capture-pipeline · scheduled ✓ inbox +12 new ✓ sent +4 new direction: outbound ✓ dedup by provider id · marked untrusted # /refresh-todo picks it up → "you replied to Karen 2d ago — no reply yet"

The honesty layer

A confident wrong answer is the expensive one.

Most assistants answer in one voice whether they know or are guessing. Charon tags each claim by how grounded it is — trust the green, check the red.

Cerberus · the installation itself

Vet what you install. Catch a poisoned dependency.

The reviews check your code; Cerberus checks the AI installation around it — the plugins, MCP servers, and dependencies you pull in. /cerberus-vet risk-assesses a third-party plugin or MCP before you install it — clones to a throwaway sandbox and scores it against a V0–V8 threat model mapped to the OWASP LLM Top-10. /cerberus-deps checks your dependencies against a live compromise registry.

Read-only, opt-in, never auto-installs. Output is risk evidence, not approval — the final call sits with your policy. Built on Joh Leonhardt's open-source engine.

# /cerberus-vet github.com/example/some-mcp clone → sandbox (purged after) V2 network egress ⚠ MEDIUM LLM03 supply chain V6 secret exposure ✓ none V8 deps vs registry ✓ clean → risk MEDIUM · 58/100 — evidence, not approval

Named agents

A small crew of standing roles — each with one job.

Some agents are named functional seats, not one-off commands. They carry work across sessions and hand off to one another — roles, never a roleplay of you.

Prometheus

/prometheus

The research seat · Titan of forethought

A standing analyst with a persistent ledger of your research beats. Each run it researches the top threads and writes a prioritised digest with framed angles — marking what's worth writing about, ready to hand to Calliope.

Surfaces and steers — never acts on its own

Calliope

/calliope

The writing seat · Muse of eloquence

Composes in your voice across modes — post, bulletin, tweet, email — taking a Prometheus angle or a raw topic to a finished draft. It loads your voice profile first, so it sounds like you, not a generic model.

Drafts only — never sends, posts, or emails

Prometheus frames a content-worthy angle and hands it to Calliope, who drafts it in your voice. You approve. A delivery seat would then send — deliberately unbuilt until the guardrails around an agent that can send are ready.

Where it gets interesting

The capabilities aren't the point — what they do together is.

Any one feature is table stakes. The value is in the chains: the parts hand off to each other, so outcomes happen that no single capability delivers.

Your inbox becomes published thinking.

Capture→Prometheus→Calliope→you approve

Your mail flows into the vault; the research seat frames a content-worthy angle from it; the writing seat drafts it in your voice. Scattered reading turns into a post — you just say yes.

Security review that just happens.

secure-code rule→3 OWASP reviewers→fp-check

Touch a code path and the doctrine fires on its own; three reviewers run in parallel; fp-check throws out what it can't reproduce. Real findings reach you before merge — without remembering to run anything.

Adopt community tools — don't fear them.

/cerberus-vet→/safe-rebuild→adopt

Vet a third-party plugin or MCP; if it's risky, safe-rebuild re-specs and re-verifies it in a scratch dir. You end up adopting a fixed version instead of rejecting a useful tool.

An adversary that knows your world.

/devils-advocate→grounded on your memory + vault

Stress-test a hard call and the critique is grounded against your notes and decisions — not generic advice. The pre-mortem raises the risks that actually apply to you.

Get started

Clone it, read it, run it, cross.

Charon runs on the Claude Code CLI. One command after cloning detects what you have, offers to install what you don't, then hands off to a ~20-minute first-run wizard.

1

Clone

Pull the public MIT repo — anywhere outside a cloud-synced folder.

2

Inspect the installer

Open install.ps1 / install.sh and skim it. Charon is a CISO tool — reading the installer before you run it is the point.

3

Run the bootstrap

It detects Python 3.10+ and Obsidian, offers auto / manual / skip per prerequisite, and creates ~/.secrets/ with restricted permissions.

4

Cross — the first-run wizard

~20 minutes to build your identity, vault, voice and framework. Then point Claude Code at the folder.

# clone the public repo
$ git clone https://github.com/acunningham-ai/Charon.git
$ cd Charon

# read it, then run the bootstrap — checks prereqs, then the wizard
$ ./install.sh            # Windows: .\install.ps1

Not in a plugin marketplace yet. Install is clone + bootstrap, not a single claude install — so first setup is a ~20-minute wizard. Marketplace packaging is near-term. We'd rather tell you than pretend the friction isn't there.
